SSO Partner Customer Frontend

Get your shop

This screen is a mock-up for the application the end customer, the merchant, is interacting with.
When the user signs up for a shop using the button below, he will be redirected to another page,
where he can log in to the MBO of his shop.
For the sake of simplicity there is no login mechanism to this page.
This simple example also assumes that the shop exists already, and ePages is correctly configured
to allow installation of an app with scope sso-read with this shop.
Of course the ePages hostname will not be supplied by the merchant either.
